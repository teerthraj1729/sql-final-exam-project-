Project Overview:-
The E-Commerce Order Management System is a database-driven project designed to manage customers, products, orders, payments, and shipping details for an online shopping platform.
This project demonstrates the practical use of SQL concepts, including database design, relationships, queries, joins, subqueries, aggregate functions, and constraints.

Objectives:-
Design a normalized relational database
Establish primary and foreign key relationships
Ensure orders are linked to customers and products
Establish relationship between payments and orders
Perform CRUD operations
Implement complex SQL queries for data analysis

Database Schema:-
The database consists of the following tables:

Categories
Products
Customers
Orders
Order_Items
Payments
Shipping


Features Implemented:-

Table creation with constraints
Insert, Update, Delete operations
WHERE, AND, OR, NOT clauses
GROUP BY and HAVING
Aggregate functions (SUM, AVG, COUNT)
Joins (INNER, LEFT, RIGHT)
Subqueries
Date & Time functions
String functions
Window functions (RANK, RUNNING TOTAL)
CASE expressions for categorization

Conclusion:-
This project demonstrates a real-world e-commerce database system using SQL.
It effectively applies relational database concepts, ensuring data accuracy, integrity, and efficient retrieval.
